Decentralized Drone Swarms

In this project, I implemented a complex Laplacian-based controller for a swarm of drones, inspired by a paper I read during my aerial robotics class at UCL. The algorithm constructs two graphs: one for accessing the drones' velocities and another for shape control. I then formulated a system of ODEs and integrated it using the fourth-order Runge-Kutta method, allowing for mid-flight formation and scale control.

UUV River Monitoring

As part of my dissertation at UCL, I chose to focus on tackling river pollution. Having lived on the coast for most of my life, I understood and saw first hand how significant an environmental issue this is. We secured a grant to purchase a BlueROV2 underwater drone. I began by implementing a Model Predictive Controller (MPC) to follow trajectories in simulation. Next, I developed a simulation to model how a pollutant from a constant source flows downstream, using FiPy, a PDE solver in Python. This simulation captured the advection and diffusion processes of the pollutant. By applying a reverse-time model, we can identify pollutant particles downstream and trace them back to the pollution source, generating a trajectory for the UUV to follow.

Unitree Go2 Path Tracking

In this project, I worked in a group to enable a Unitree Go2 robot to follow predefined trajectories. I began with state estimation, specifically estimating roll, pitch, and yaw using data from the onboard IMU. To achieve this, I implemented a Kalman filter to fuse sensor data and estimate the robot’s orientation. One of the main challenges was yaw estimation, which is typically derived from a magnetometer, GPS, or visual input. However, as part of the challenge, we were not allowed to use any of these sensors. As a workaround, we assumed a constant yaw throughout each run. Given that the robot was operating on flat ground, executing relatively short trajectories, and had reliable odometry, this assumption was reasonable and allowed us to proceed. After establishing orientation estimates, we implemented a Model Predictive Controller (MPC) based on a unicycle model to track the given trajectories. We tested several paths, with the most complex being a figure-eight trajectory.

Pick and Place

I implemented a custom pipeline to grasp and place square and cross-shaped objects into a box using point cloud data from a depth camera. After clustering to find object centers, I applied corner detection for orientation and used a rotation formula for grasp point calculation. Shape classification was handled via centroid properties. The entire task was executed using the MoveIt library.

CartPole MPC

In this project, I used Model Predictive Control (MPC) for the CartPole environment using OpenAI's Gymnasium. The objective was to keep the pole balanced on the cart without it tipping over.
